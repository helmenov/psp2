# Pythonプロジェクト管理アプリ

Python言語のプログラミングの目的は，以下の2種類と考えられます．

これらを「プロジェクト」といいます．

1. アプリケーションを作る
    - `> python script.py` で実行するもの（CLIやGUIでインタフェースが起動したり，密かに待ち受けしたり（サーバー））
        - ↑ の `script.py` のような起動用ファイルを「エントリー・ポイント」と云う．
    - このエントリポイントとなるソースコードを書く．
      - そのソースコードから他のソースコードを読み込むこともある．（複数のファイルに分割することもある）
    - 他人に配布するとしたら，実行ファイル．（ソースコードは普通渡さない）
2. パッケージを作る
    - ソースコードの中で参照されるフォルダを構成する．
        - このフォルダ名を「パッケージ」と云う．
        - フォルダの中に，様々な関数定義が書かれた .py ファイル（モジュール）を持つ．
        - モジュールは `> python module.py` で実行されない（読まれるだけで即座に終了してしまう）
    - そもそも配布するもの
        - `pip install pkg` でインストールして使う．
    - 複数のアプリを配布したいときも「パッケージ」．

結局，パッケージでアプリケーションを配布することもあるので，違いが不明確ですが，

- 当初から配布するつもりのときは，「パッケージ」
- 配布するつもりが無いときは，「アプリケーション」

と考えておきましょう．

しかし大概は，「パッケージ」プロジェクトで始めると思われます．配布するつもりで始めて「やっぱり辞めた」で配布しなくてもよいわけなので．

## *Poetry*

`poetry` は，「パッケージ」プロジェクトを管理するソフトウェアです．

1. 新しくパッケージの開発を始める

    1. 開発用のフォルダに移動
        ```
        > cd （開発用フォルダ）
        ```
    2. パッケージ開発を始める（プロジェクト名:`my-package`, パッケージは`mypackage`とする）

        ```
        > dir
        (空っぽだったり，他のプロジェクトがあったり)
        ```

        プロジェクトフォルダ作成

        ```
        > mkdir my-package
        ```
        Visual Studio Code でプロジェクトフォルダを開く．
        ```
        > code my-package
        ```

        Visual Studio Code のメニューからターミナルを立ち上げて，Poetry でパッケージ構成に設定

        ```
        > poetry new .  --interactive
        ```
        以下のように色々質問されて，構成ファイル `pyproject.toml` やフォルダのテンプレート構造が構成される．
        ```
        Package name [my-package]:
        ```
        の質問は，パッケージ名を尋ねています．「：」の後ろに答えてください．四角括弧は何も答え無かった場合にパッケージ名とみなされるものです．（現在のフォルダ名が提示されています）

        フォルダ名（プロジェクト名）は `my-package` でしたが，パッケージの名前は `mypackage` にしたいので，そのように答えてください．

        あとは，てきとーに．暇なら読んで答えて．

        処理が終わると，結局，以下のようなフォルダ構成のフォルダ（プロジェクト）が作成される．
        ```
        my-package/ # <- プロジェクトルート（根っこ）
        ├── pyproject.toml # <- プロジェクトの設計書
        ├── README.md
        └── src/
            └── mypackage/ # <- このフォルダがパッケージ
                └── __init__.py
        ````
        プロジェクトの設計書 `pyproject.toml` には色々書かれますが，大事なのは，

        ```
        [tool.poetry]
        packages = [{include = "mypackage", from = "src"}]
        ```

        と書いてあることです．配布するパッケージは，`mypackage` というフォルダで，プロジェクトルートの下の `src` というフォルダの下にある，と云う意味です．

        この時点では，`mypackage` フォルダには，`__init__.py` というファイルが1つあるだけですが，

        このフォルダに，次々とソースコードを追加していきましょう．
